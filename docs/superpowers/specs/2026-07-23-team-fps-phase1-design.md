# CQJC-3D 团队竞技枪战 · 第一期设计（普通团队竞技）

> 状态：已通过头脑风暴，待用户复核
> 日期：2026-07-23
> 基础：现有 three.js 第一人称校园漫游（src/ ~1300 行）

## 1. 背景与目标

把现有的校园漫游改造成**团队竞技第一人称枪战游戏**。

**故事设定**：1947 年国共内战正酣，两军在战场殊死相搏之际，一道时空裂隙将两支军队连人带枪卷入这座现代高职校园，战斗在此地继续。此设定圆满解释了"为何两军在现代校园开战"。

**第一期只做「普通团队竞技」**：8v8，红蓝阵营，固定点复活，哪队击杀数**先到 50 获胜**。玩家可**自选加入国军（蓝方）或共军（红方）**。

**已确定的核心约束**：
1. **单机 + 人机优先**：第一期不做真实联机，但完整实现"匹配 UI → 20 秒 → 填人机"流程，并预留网络层接口。联机为第二期。
2. **美术**：通用 CC0 士兵模型（Mixamo / Quaternius 等），红/蓝涂装区分阵营。历史军服不做。
3. **手机优先，必须流畅**：16 个带骨骼动画 + AI 的角色同场是最大技术挑战，优化为第一优先级。

## 2. 分期路线

- **第一期（本 spec）**：完整可玩的单机团队竞技对局，可出 APK。
- **第二期**：网络层（WebRTC / relay），替换状态层的权威来源，接入真实匹配。
- **第三期**：更多模式（战术竞技 / 占点 / 攻防），走规则层插槽；精细美术。

**解耦原则**：游戏逻辑与「网络」「美术」彻底分离，使二三期是叠加而非重写。

## 3. 架构分层

```
状态层  GameState      —— 唯一权威数据源(玩家/机器人/击杀数/阵营/比分)
                          抽象为"本地权威"，二期换"网络权威"只改这一层
实体层  Combatant      —— 玩家与机器人共用(血量/开火/受击/死亡/复活)
AI 层   BotBrain       —— 状态机 + 手机分帧调度
战斗层  Weapon/Combat  —— 5 把枪(射线命中)+ 伤害 + 固定点复活
规则层  Mode           —— DeathmatchMode(8v8,先到50杀);可插拔,为占点/攻防预留
演出层  Intro          —— 开场故事旁白(浏览器TTS)+ 地图扫描运镜 + 跳过
流程层  MatchFlow      —— 阵营选择 → 匹配(预计20秒→填人机) → 出生 → 对局 → 结算
复用    player.js(改第一人称持枪) / campus.js / touch.js / hud.js / sky.js / audio.js
```

每层通过明确接口通信，可独立理解与测试。

## 4. 各子系统细节

### 4.1 开场演出（Intro，可跳过）
- 黑屏起，`SpeechSynthesis` 中文旁白（配低沉背景音）：
  > "1947 年，内战正酣。国共两军在战场殊死相搏……一道诡异的时空裂隙骤然撕开，两支军队连人带枪，被卷入了这座陌生的校园。硝烟未散，战斗，在此地继续。"
- 旁白同时，摄像机高空俯瞰做**地图扫描运镜**（缓慢平移 + 下压，掠过校园地标），复用现有光照。
- 右下角"跳过 ▶"按钮，跳过直接进阵营选择。

### 4.2 阵营选择 + 匹配流程（MatchFlow）
1. **阵营选择**：国军（蓝）/ 共军（红）二选一。
2. **匹配界面**：显示"预计匹配时间 20 秒" + 进度条。
3. 20 秒（或凑够真人时）满员 → 未满位置**填人机**，两队各补满至 8 人。
4. 3 秒出生倒计时 → 进场。
- 第一期背后全为人机，但流程与二期联机一致。

### 4.3 战斗手感（Combat）
- **第一人称持枪**：屏幕下方 viewmodel，含开火后坐、射击间隔、换弹动画。
- **命中判定**：射线检测（hitscan）；躯干普通伤害，爆头 1.5×。
- **血量**：100；受击有方向指示 + 红屏反馈；脱战 5 秒后缓回血。
- **死亡 → 固定点复活**：各阵营 2~3 个固定出生点；死后 3 秒在己方点复活。
- **操控**：桌面 WASD + 鼠标；手机 摇杆移动 + 右半屏滑动转视角 + 开火按钮（复用/扩展 touch.js）。

### 4.4 枪械（Weapon，5 把，属性差异化）

| 枪 | 伤害 | 射速 | 后坐 | 有效射程 | 定位 |
|---|---|---|---|---|---|
| 手枪 | 中 | 中 | 低 | 近中 | 副武器/保底 |
| 步枪 | 中高 | 中 | 中 | 中远 | 全能主力（默认） |
| 冲锋枪 | 低 | 极高 | 中 | 近 | 突进压制 |
| 狙击 | 极高(爆头秒) | 慢 | 高 | 远 | 架点 |
| 霰弹 | 近距极高 | 慢 | 高 | 极近 | 巷战 |

- 出生时选枪（默认步枪），死亡复活时可换。
- 数值以配置表形式存放（damage/fireRate/recoil/range/spread/magSize），便于调平衡与扩展。

### 4.5 人机 AI（BotBrain）
- **状态机**：巡逻 → 发现敌人 → 交火（走位 + 间歇射击）→ 血少找掩体 → 死亡 → 复活。
- **性能三招**：
  1. AI 逻辑**分帧调度**——每帧只更新一部分机器人（budget 分摊）。
  2. 远处机器人**动画降频/停更**。
  3. **距离 LOD**——远处简化模型与逻辑。
- **难度参数**：命中率、反应时间、走位幅度可调；默认"能打但不变态"，保证单人可玩性。

## 5. 手机性能预算（16 人同场）
- 共享骨骼 / 尽量合批；动画更新按距离与可见性降频。
- 射线命中优先用 three-mesh-bvh（项目已依赖）加速。
- AI tick 分帧，单帧总预算封顶。
- 远处角色 LOD / imposter。
- 目标：中端手机稳定可玩帧率，16 人不卡。

## 6. 复用与改造现有代码
- `player.js`：由漫游改为第一人称持枪 Combatant 的控制外壳（保留胶囊碰撞/移动）。
- `campus.js` / `sky.js`：地图与光照直接复用。
- `touch.js`：扩展开火/换枪/转视角控件。
- `hud.js`：扩展为战斗 HUD（血量/弹药/比分/击杀提示/准星）。
- `audio.js`：扩展枪声/受击/脚步/旁白背景音。

## 7. 测试策略
- 纯逻辑层（GameState / Weapon 伤害 / Mode 胜负判定 / BotBrain 状态转移）以 node 单元测试覆盖，不依赖渲染。
- 沿用现有 playwright 冒烟测试（walk-test / touch-test 思路）验证进场与基本操作不崩。

## 8. 明确不做（YAGNI，第一期）
- 真实联机 / 内网穿透（第二期）。
- 战术竞技 / 占点 / 攻防模式（第三期，但 Mode 层预留插槽）。
- 历史写实军服建模。
- 排位 / 账号 / 结算存档。

## 9. 可扩展性预留
- **网络**：GameState 权威来源可替换（本地 ↔ 网络）。
- **模式**：Mode 为接口，DeathmatchMode 是首个实现。
- **武器**：配置表驱动，加枪即加数据。
- **阵营/美术**：涂装与模型可换，不影响逻辑。
