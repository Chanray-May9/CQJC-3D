# 人机 AI Implementation Plan（子计划 3/5）

> REQUIRED SUB-SKILL: superpowers:executing-plans。

**Goal:** 让占位敌人变成会巡逻、索敌、走位交火、血少撤退、朝玩家开枪的机器人；难度"比真人稍强但不秒杀"，手机上分帧调度保流畅。

**Architecture:** 纯逻辑 `botBrain.js`（有限状态机 + 感知输入→动作输出，TDD 单测）与集成 `botController.js`（驱动 avatar 沿 BVH 移动、朝向、对玩家做 LOS 射线与命中概率射击）。arena 把每个红队敌人换成 bot；玩家可被打死并复活。借鉴通用做法：反应延迟(reaction time)、随距离/移动衰减的命中概率、偏好交战距离、分帧 AI 预算。

**难度取向（比真人稍强，不秒杀）：**
- 发现玩家后有 0.35–0.55s 反应延迟才首次开火。
- 命中为概率模型：近距高、远距低、玩家移动再打折；命中才结算武器伤害，未命中不结算。
- 单发伤害用武器表(步枪26)，配射速——击杀需连续命中数百 ms~1s，给玩家反应与还手空间。
- 血量<30% 撤退找远离玩家方向，脱战回血。

---

### Task 1: BotBrain 状态机（TDD）
**Files:** Create `src/game/botBrain.js`, Test `test/botBrain.test.js`
- [ ] TDD。输入 `sense={ dt, now, distance, hasLOS, playerAlive, health }`，输出 `think(sense)→{ state, wantShoot, moveMode, aimError }`。覆盖：无 LOS→PATROL 不开火；获得 LOS 后未过反应延迟→不开火；过延迟且在射程→wantShoot；health 低→RETREAT；playerAlive=false→PATROL。反应延迟、交战/撤退阈值可配置。

### Task 2: 命中概率模型（TDD）
**Files:** `src/game/botBrain.js`（同文件加导出函数）, `test/botBrain.test.js`
- [ ] TDD。`hitChance({ distance, playerMoving, difficulty })→0..1`：近距高(≤0.9)、随距离线性降、玩家移动乘 0.6~0.75、difficulty 缩放。断言单调性与边界钳制(0.05..0.95)，绝不 1.0（不必中/不秒杀）。

### Task 3: BotController 集成
**Files:** Create `src/combat/botController.js`
- [ ] 每个 bot 持有 Combatant+Avatar+BotBrain+WeaponRuntime。`update(dt, ctx)`：对玩家做 LOS 射线(BVH)、算 distance、喂 think；按 moveMode 产生移动意图(巡逻 waypoint / 交战保持中距 strafe / 撤退远离)，沿 BVH 推移+与其它 bot 分离，写回 avatar 位置与朝向(faceYaw 面向移动或玩家)；wantShoot 且武器就绪→按 hitChance 掷骰，命中则对玩家 Combatant 结算伤害并触发受击回调。死亡走倒地动画，复活归位。

### Task 4: 接入 arena + 玩家可死亡
**Files:** Modify `src/combat/arena.js`, `src/main.js`, `src/hud.js`, `index.html`
- [ ] arena 用 BotController 替换静止敌人；`update` 里 tick bots(分帧：每帧只更新一批，预算封顶)；玩家受击→红屏/方向指示，血量到 0→死亡计时 3s 在己方出生点复活。玩家死亡横幅"阵亡·复活中"。onKill 支持敌方击杀玩家给红队+1。

### Task 5: 冒烟 + 调优 + 合并
- [ ] smoke：bot 会朝玩家靠近并在数秒内造成伤害但不瞬秒；玩家能被打死并复活；16 bot 帧率达标(截 __debugHealth fps)。单测全绿→合并 main。
